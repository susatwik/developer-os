# Security Policy

## Supported Versions

Developer OS follows the latest `main` branch and the most recent tagged release.

## Reporting a Vulnerability

If you discover a security issue:

1. Do not open a public issue with exploit details.
2. Contact the maintainers privately through the repository security contact or issue triage channel.
3. Include:
   - A short description of the issue
   - Affected components
   - Steps to reproduce
   - Any mitigation ideas

## Security Expectations

- Store credentials in environment variables or secret managers.
- Do not commit tokens, cookies, or private API keys.
- Prefer least-privilege access for GitHub and LeetCode integrations.
